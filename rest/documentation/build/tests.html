send_volatile_request(method, endpoint, data, access_token, status_code=200, value=None)[source]¶

Send a volatile request to the backend. A volatile request is one whose return value is expected to change over time. The request is retried until the desired value is returned, or until some time passes.

Parameters

  • method (str) – The method to use when sending the request - usually POST or GET.

  • endpoint (str) – The endpoint to which to make the request.

  • data (dict) – The data to send along with the request.

  • access_token (str) – The access token to use to retrieve protected resources.

  • status_code (int) – The expected status that the function should return. The status corresponds to an HTTP status code. By default, the function expects the request to succeed.

  • value – The desired return value.

Type

value

Returns

The response.

Return type

requests.models.Response

tests.test.retry(times=10)[source]¶

A decorator function that keeps trying to get a result until it times out. The function is useful in cases where results are not available immediately. This case arises when the blockchain needs to be mined before getting a result.

Parameters

times (int) – The number of times that the function should be retried. If the desired status is never received, the function returns the last output.

Returns

The wrapped function.
