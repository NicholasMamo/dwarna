Handlers¶

The route handler encapsulates the REST API. Its job is to handle the given requests.

class biobank.handlers.handler.PostgreSQLRouteHandler(connector, blockchain_connector, *args, **kwargs)[source]¶

This class is based on PostgreSQL. It assumes that all the data is stored in a PostgerSQL database.

The general workflow of handlers is as follows:

  1. Create an empty response.
  2. Extract the given arguments, if any, and sanitize them.
  3. Perform any required validation, raising exceptions if anything fails.
  4. Communicate any queries with the database, if need be, and fetch a status.
  5. Set the status code and write any required output to the response body.

Hyperledger Handler¶

An implementation that interfaces with Hyperledger Composer’s own REST API.

class biobank.handlers.blockchain.api.hyperledger.HyperledgerAPI(host, default_admin_port, default_multiuser_port, connector)[source]¶

The Hyperledger API is equipped to interface with Hyperledger Composer’s REST API.

It can communicate with two REST APIs at the same time:

  1. The admin API; and
  2. The multi-user API.

All multi-user functions require an access token to operate. When users authorize their profile, they get a card. They import the card into Hyperledger, which creates a new card with credentials. Users can export this card and save it to re-import it later.

At the same time, an access token is generated for them. This access token allows access to this API.

Variables:
  • _host (str) – The hostname where the Hyperledger Composer REST API is running.
  • _default_admin_port (int) – The default port where admin requests are made.
  • _default_multiuser_port (int) – The default port where multi-user requests are made.
  • _connector (connection.connection.Connection) – The connector that is used to access the data store.
