Variables

routes (dict) –

A list of routes that are accepted by the REST API. Associated with each route are a number of methods that it accepts. Each method has:

  • the handler class;

  • the handler function;

  • a list of scopes;

  • the required parameters; and

  • a boolean indicating whether the function is personal.

Scopes need not be unique. For example, different ways of removing participants may share the same scope.

A personal function allows an individual to access and manage their own data.

The resource server receives requests from an application and services them. An important component of the resource server is its validation of requests.

Variables

  • _routes (dict of dicts) – A list of routes, associated handles and scopes

  • _route_handlers (dict) – The objects that are used to handle requests for different routes.

__init__(access_token_store, auth_code_store, client_store, token_generator, routes, route_handlers)[source]¶

Create the resource server based on oauth2.Provider. The first arguments should be the oauth2.Provider’s.

The resource server receives a list of routes that it can handle. Routes are dictionaries, where the keys are the paths. The keys themselves are split into the methods that they accept. This method is another dictionary of the form:

{
     "/participant": {
             "POST": {
                     "handler": handler_class,
                     "function": create_participant,
                     "scopes": ['create_participant'],
                     "parameters": ['username'],
                     "self_only": True
             }
     }
}

Associated with each route is the function that handles it. Each route also has a list of scopes that restrict access to protected resources. Moreover, to protect against erroneous requests, the method is linked with the API endpoint as well.

The self_only attribute is optional. When set to True, it restricts users to access only their own data. To ensure that the user is accessing their own data, the access token owner is checked with the username in question.

The provided route handler contains the functions that handle each route.

Parameters

  • access_token_store (oauth2.store.AccessTokenStore) – Store for all access tokens.

  • auth_token_store (oauth2.store.AccessTokenStore) – Store for all authorization tokens.

  • client_store (oauth2.store.memory.ClientStore) – Store for the known clients.

  • token_generator (oauth2.tokengenerator.TokenGenerator) – Generates new authorization and access tokens.

  • routes (dict of dicts) – A list of routes, associated handles and scopes

  • route_handlers (dict) – The objects that are used to handle requests for different routes.
